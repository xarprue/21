Comunicación y seguridad.

En una aplicación web, la comunicación entre el cliente y el servidor es clave. Esto se realiza generalmente a través de solicitudes HTTP, y las respuestas suelen estar en formato JSON o XML. Para mejorar la seguridad, se implementan técnicas como:

1.- HTTPS: Para encriptar las comunicaciones y evitar que sean interceptadas.


2.- Autenticación y Autorización: A través de tokens como JWT (JSON Web Token) para asegurar que solo los usuarios autorizados puedan acceder a ciertos recursos.


3.- CORS (Cross-Origin Resource Sharing): Para controlar qué dominios pueden interactuar con la API.
